# スクショ壁打ちモード

robro-jp のドキュメント系ページ（`/guide` 等）に追加するスクリーンショットを、Yuki と対話しながら決めます。

## 振る舞い

1. まず Yuki に「どのページ／どの機能のスクショか」を尋ねる（引数 `$ARGUMENTS` で指定があればそちらを優先）
2. 対象が決まったら、以下を具体的に一緒に詰める:
   - **画面URL**（例: `https://robro.fortunep.net/game/123456` の特定状態）
   - **状態**（ログイン済/未ログイン、モーダル開いた状態、データの例、など）
   - **トリミング範囲**（画面全体 / 特定セクション / モバイル幅 / PC幅）
   - **注釈の有無**（矢印、囲み、伏字が必要か）
   - **保存先ファイル名**（`public/guide/step-google-signup.png` のような）
   - **差し込み箇所**（例: `app/guide/page.tsx` の ② Google アカウント作成セクション）
3. 撮影後、Yuki が画像を配置したら `Image` コンポーネントで差し込むコードを書く
4. alt テキストを決める（視覚障害者・読み上げ向け）

## ルール

- Yuki がまだ撮ってない段階では「こういう画面のこの状態を撮って」と具体的に指示する
- 撮影後の配置は `next/image` を使う。`public/guide/` 配下に画像を置く
- ファイル名は `step-<section-slug>-<n>.png` 形式（例: `step-login-1.png`）
- プライバシー配慮: スクショに個人情報（メールアドレス、アイコン等）が含まれる場合は伏字を提案

## 引数

$ARGUMENTS があればそれを壁打ち対象ページとみなす（例: `/screen /guide のタグ投票セクション`）。
